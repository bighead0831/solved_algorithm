🧠 Java 자료구조 마스터: Hash & Map 정복하기
작성일: 2026년 3월 26일

학습 목표: 해시(Hash)의 원리를 이해하고 Java 컬렉션 프레임워크의 구조를 바로잡는다.

1. 🔍 Hash: 클래스가 아닌 '개념'이자 '알고리즘'
해시는 특정 도구나 함수라기보다, 데이터를 다루는 핵심 원리입니다.

정의: 임의의 길이를 가진 데이터를 고정된 길이의 고유한 값(해시코드)으로 변환하는 알고리즘입니다.

역할: 데이터의 **'디지털 지문'**을 만들어 아주 빠른 속도로 데이터를 찾거나 중복을 확인할 때 사용합니다.

Java에서의 활용: 모든 객체는 hashCode() 메서드를 통해 자신의 지문(해시값)을 반환할 수 있습니다.

2. 🧱 Interface vs Class: 구조 바로잡기
우리가 사용하는 HashSet, HashMap 등은 '메서드'가 아니라, 인터페이스를 구현한 **'클래스'**입니다.

3. 🛠️ 주요 자료구조 핵심 요약 (교정본)
✅ HashSet (중복 제거의 마술사)

오해: 요소의 존재 여부만 알 수 있고 중복 여부는 모른다? (X)

진실: 중복을 절대 허용하지 않는 집합입니다. 똑같은 데이터를 넣으려고 하면 해시 알고리즘이 "이미 있네!" 하고 막아버립니다. 중복 제거가 주 목적일 때 사용합니다.

✅ HashMap (이름표 관리자)

특징: Key와 Value를 짝지어 저장합니다.

성능: 해시 알고리즘 덕분에 데이터가 아무리 많아도 Key값만 알면 빛의 속도로 Value를 찾을 수 있습니다.

단점: 저장 순서나 정렬 상태를 전혀 보장하지 않습니다.

✅ TreeMap (똑똑한 정렬가)

특징: HashMap의 기능을 다 가지면서, 추가로 **Key값을 기준으로 자동 정렬(오름차순)**해줍니다.

용도: "사전순으로 정렬해서 출력해라"라는 조건이 붙은 알고리즘 문제에서 최고의 효율을 발휘합니다.
