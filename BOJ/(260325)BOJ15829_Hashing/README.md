2. 자료형의 정밀도와 메모리 구조 (Bit vs Byte)

단위의 명확화: * int: 4바이트 (32비트), 약 ±21억까지 표현.

long: 8바이트 (64비트), 약 ±900경까지 표현.

데이터 타입 선택: 합계가 21억을 넘어갈 가능성이 있는 계산(예: 대용량 해시 합산)에서는 반드시 **64비트(long)**를 사용해야 함을 체득.

3. 모듈러 연산(%)의 성질과 안전밸브

분배 법칙: (A+B)(modM)=((A(modM))+(B(modM)))(modM).

안전밸브 역할: 중간 계산 과정에서 값이 너무 커져 오버플로우(Overflow)가 발생하는 것을 막기 위해, 매 덧셈과 곱셈 직후에 % M 연산을 수행하는 습관의 중요성 파악.
